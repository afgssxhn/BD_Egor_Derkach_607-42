# Индивидуальный проект: Библиотека

## Сценарий

Городская публичная библиотека. В системе хранятся: каталог книг, карточки читателей, выдачи книг, брони, штрафы за просрочку. Работают с системой читатели (через личный кабинет) и сотрудники разных уровней.

## Роли и что они делают

| Роль | Кто | Что может |
|---|---|---|
| `lib_reader` | зарегистрированный читатель | искать книги в каталоге, смотреть свою карточку / свои выдачи / свои брони / свои штрафы, создавать бронь |
| `lib_librarian` | рядовой библиотекарь | всё что читатель + выдавать/принимать книги, управлять бронями, искать читателей по билету |
| `lib_senior` | старший библиотекарь | + управлять каталогом книг (добавлять/править/списывать), выставлять и списывать штрафы |
| `lib_admin` | администратор системы | полный доступ: управление читателями (регистрация/блокировка), сотрудниками, DDL |

Иерархия наследования: `admin → senior → librarian → reader`. Гостя (как в задачной БД) тут нет — в библиотеке система не работает с анонимами, каталог публично читают через веб, а в БД попадаешь только после регистрации.

## Соответствие моделей доступа

- **RBAC** — основной каркас: 4 роли с наследованием, выдача прав ролям, не юзерам.
- **DAC-элементы** — книга привязана к «owner_id» (старший библиотекарь, который внёс книгу): автор записи может её править без дополнительных настроек. Это не чистый DAC, скорее гибрид.
- **Колоночное ограничение** — у читателя нельзя смотреть чужие email и hashed_password — это колоночный GRANT на роли librarian/senior.
- **RLS** — фильтрация «свои выдачи», «свои брони», «свои штрафы» через политики с `current_user`.
